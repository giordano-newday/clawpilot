## Directory-specific instructions

- `docs/clawpilot-project.md` is the high-level product and architecture spec. Keep naming, architecture, and intent consistent there first.
- `docs/plans/` contains human-written plans. Keep them concise, current, and explicit about trade-offs and verification.
- `docs/reports/` is generated output. Do not hand-edit it unless the task is specifically about generated report assets.
- Avoid duplicating the same instruction or architecture text across plan files. Link or reference the canonical spec instead.
- When documentation and code diverge, update the source-of-truth doc rather than adding contradictory notes elsewhere.
